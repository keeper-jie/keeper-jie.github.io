1.DLL调用

创建和使用你自己的动态库

主要包括以下四个内容：

1）Create a DLL project in Visual Studio.

2）Add exported functions and variables to the DLL.

3）Create a console app project in Visual Studio.

4）Use the functions and variables imported from the DLL in the console app.

5）Run the completed app.

我的IDE是VS2017

__declspec(dllexport)修饰符告诉编译器（compiler）和链接器（linker）从DLL中导出函数或变量

__declspec(dllimport)修饰符优化应用程序中导入的函数或变量

1.1 MathLibrary.h 官网代码中extern "C"作用：指示编译器这部分代码按C语言的进行编译，而不是C++的。

由于C++支持函数重载，因此编译器编译函数的过程中会将函数的参数类型也加到编译后的代码中，而不仅仅是函数名；而C语言并不支持函数重载，因此编译C语言代码的函数时不会带上函数的参数类型，一般只包括函数名。

例如，假设某个函数的原型为：void foo( int x, int y);该函数被C编译器编译后在符号库中的名字为_foo，而C++编译器则会产生像_foo_int_int之类的名字

1.2 生成的dll文件和lib文件在项目中的DEGUG目录中

1.3 在使用dll操作中，把dll文件复制项目中DEBUG目录下和项目cpp文件同目录下均可。放在其他地方可能提示缺少dll文件，为了解决这个问题官网上面有对应操作（在编译后事件中添加dll文件，在该命令xcopy /y /d "..\..\MathLibrary\$(IntDir)MathLibrary.dll" "$(OutDir)"中一定要注意两个引号之间空格），可见在官方示例中是放在DEBUG目录下。

1.4 给我的启示：找问题最好在官网找资料，效率最高（之前参考CSDN什么的浪费了很多时间，还没有弄好）
